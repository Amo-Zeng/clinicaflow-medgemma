# ClinicaFlow 冠军冲刺计划（Sprint）

目标：把“case_reports 真实来源回归 + 可审计安全闸 + MedGemma 推理”做成评审一眼能感知价值的**产品级作品**（不做虚假临床声明）。

> 原则：只使用**合规公开来源**（PMC/CDC/NIH/NHS/MedlinePlus 等），所有“真实案例”均为**去标识化改写 vignette**，不包含真实患者数据/PHI。

---

## Sprint Backlog（按顺序冲刺）

### 1) 让 `case_reports` 变成“可见证据”（UI + 导出）

- [x] Console/Streamlit：在加载 vignette 时展示 `source.title + source.url + note`，支持一键打开链接
- [x] 导出物：`audit bundle` / `judge pack.zip` / `note.md` / `report.html` 中包含 `case_meta.json`（vignette set/id + source + rationale）

验收：评审在 UI/导出里无需翻代码就能看到“这条 vignette 来自哪里”。

### 2) Safety rulebook 加权威引用（rulebook → evidence）

- [x] `safety_rules_catalog()` 为每条 safety trigger / vitals trigger 增加 1–2 条权威证据链接
- [x] Rules/Governance UI 可点击查看引用（不需要外部 API key）

验收：Rules 页每条 trigger 都能看到可点击证据链接，且导出 `resources/safety_rules.json` 同步包含引用。

### 3) 扩充 `case_reports` 到 ~50，并补“非典型/同义词覆盖”

- [x] 覆盖 cardiopulmonary / neuro / GI bleed / OB / sepsis / syncope 各方向更多 open‑access case reports
- [x] 增加“mimic/非典型表现”用例（例如 stroke mimic、PE 非典型、GI bleed 早期等）

验收：`python -m clinicaflow.benchmarks.vignettes --set case_reports --print-markdown` n≈50 且每条都有来源链接。

### 4) 增加 Ablation（证明 agentic workflow “不可替代”）

- [x] 同一套 vignettes：Baseline vs Text‑only/Reasoning‑only vs Safety‑only vs Full pipeline
- [x] 产出可复现 markdown 表格，写入 Kaggle writeup/冠军 writeup

验收：一条命令生成 ablation 表格，并且写入 submission pack。

### 5) Clinician review 工具链（不虚构；有则加分）

- [x] 提供 review 工具链：Console UI 的 Review tab + `clinicaflow benchmark review_packet` + `review_summary`，并内置“无 review 不声明”的护栏
- [x] writeup/模板：`docs/CLINICIAN_REVIEW_TEMPLATE.md` + writeup 中明确说明当前是否包含真实 review
- [ ] （可选，但强烈建议）让 1 位临床人员（或护理/PA）评 5–10 个 vignette，导出并保存为 `reviews/clinician_reviews.json`

验收：
- 若有真实 review：writeup 中出现可审计的 reviewer 反馈摘要（无 PHI）。
- 若无真实 review：writeup 明确写出“未进行 clinician review，因此不报告 clinician feedback”（不虚构）。
